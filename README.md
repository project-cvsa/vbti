# VBTI

## 技术栈

React Router + shacn/ui + jotai

## 算法

这简直是我见过最抽象的需求。。。既要又要的

我决定摒弃先前版本基于性格（MBTI）向量相似度+特判的思路。
因为实际上，对于同一个MBTI的不同角色，我们很难给他们赋予不同的向量，还要保证结果分布合理。
（因为能确定一个角色的MBTI就够不容易了，还要在此基础上确定它具体的偏向太无理了）

综上，我选用了基于概率分布的算法。步骤如下：

1. 核心函数在运行前，先生成一个所有角色概率相等的概率分布。
2. 定义角色偏好强化函数：该函数根据一些特殊题目（如3, 5, 10, 29, 39等）中的答案情况，修改对应角色的概率。
3. 如果33和34题中有一个选择的是D（听中V or 听日V），直接将另一个语言的所有角色概率降为0。
4. 如果第0题中测试者希望“测出与我MBTI接近的”，就将不符合其MBTI的角色概率降为0；
	如果测试者希望“和我爱听的术力口一致，是我推就最好”，就执行2；
	如果测试者希望“测出小众的歌姬”，就将标记为冷门的歌姬概率提高；
	如果测试者没有明显偏好，则强化与其MBTI相符的角色的概率、执行2，并适当提高热门歌姬的概率。
5. 将最终得到的概率分布归一化后，以用户的MBTI性格向量作为输入，送入一个四维柏林噪声函数，将该函数的输出作为随机源，得到概率分布的最终采样。
	> 使用用户MBTI性格向量与柏林噪声的原因是，这样可以使相近的用户得到的结果更相近，同时确保相同输入可以得到相同输出


附（第五步的具体实现原理）：
1. 将MBTI向量（E, S, T, J）作为噪声的输入坐标：比如 Noise(E, S, T, J)。
2. 用噪声函数生成一个连续的、确定性的伪随机数。
3. 将最终的概率分布（[P_绫, P_言, P_初音, ...]）计算成累积分布（[0.2, 0.5, 0.8, ..., 1.0]）。
4. 用伪随机数，从累积分布函数（CDF）中采样。看随机值落在累积分布的哪个区间，就选中哪个角色。
